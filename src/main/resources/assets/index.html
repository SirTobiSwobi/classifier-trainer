C3 Classifier Trainer API

This microservice is an abstract implementation of the C3 Classifier-Trainer API. Microservices are based on it.

  • Get Started

/metadata

Renders the metadata returned by the service.

/documents

Here, all documents relevant for the classifier trainer can be created, read, updated and deleted.

/categories

Here, all categories relevant for the classifier trainer can be created, read, updated and deleted.

/relationships

Categories can form relationships. This means, that certain categories can be sub-categories of others. They also can be equivalent to each other. This has logical implications for assigned documents. Documents assigned to sub-categories are always implicitly also assigned to their super categories as well as to their equivalent categories. The service automatically computes these relationships. Here you can edit these relationships.

/targetfunction

In Text Categorization, the target function is the manually created definition which document belongs to which category. The trainer needs this to learn from.

/configuration

Depending on the utilized machine learning algorithm, different hyper-parameters for the learning algorithm can be specified. This happens using this API endpoint.

/models

Models are generated, when the trainer learns a representation from the existing examples. These models can then be shared with other athlete services that implement the same model. Generating models triggers training,

/evaluations

During every training, a self-evaluation is performed. This measures how well the system performed during training. This endpoint can be used to choose the appropriate model for a given task.

/model

Every trainer service can also serve as athlete service. Every athlete service needs to select one active model that is used for every categorization.

/categorizations

This endpoint is used to perform categorizations of previously unknown documents using the active model of the classifier.

/retraining

If categories and their relationships are changed, training a new model reflecting these changes is advisable. This endpoint shows wether a retraining is required or not.

Classifier-Trainer

A classifier assigns documents to categories based on their content by employing machine learning. This service is part of the Classifier Trainer/Athlete pattern developed by Tobias Eljasik-Swoboda. A Trainer Service uses machine learning to learn from existing document-to-category assignments. After having learned from these examples, an Athlete Service can use these insights to perform classifications. This facilitates Scale-Out. A Trainer Service always includes an Athlete Service. Each C3 Service is available as JAR file and Docker Container. This makes them runable anywhere.
The service employs multiple endpoints using a REST/JSON API. This is wrapped in the classifier-trainer.js JavaScript employed by this web GUI.
Serving this web GUI can be turned off in the configuration yml prior to starting the service.
